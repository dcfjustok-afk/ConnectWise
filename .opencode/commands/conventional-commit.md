description: 生成约定式提交信息（冒号后中文）
---
请基于当前仓库变更执行以下流程：

1. 读取当前 staged + unstaged 的 git 变更。
2. 判断改动是否满足原子化提交原则（一个提交只做一件事）。
3. 如果不满足，先给出拆分建议（按提交粒度分组）。
4. 为每组改动生成 1 条候选提交信息，格式必须是 Conventional Commits：
   - `type(scope): 中文描述`
   - scope 不明确可省略为 `type: 中文描述`
5. 严格要求：冒号 `:` 后面的描述必须是中文。
6. 在用户明确要求“直接提交并推送”时，继续执行：
   - 先给出推荐提交信息并确认原子化分组。
   - 执行 `git add`（仅添加本次目标文件）。
   - 执行 `git commit -m "type(scope): 中文描述"`。
   - 执行 `git push` 推送到当前分支对应远端。
   - 返回提交哈希与远端分支信息。

输出格式：
- 原子化评估：通过/不通过（含原因）
- 拆分建议：
- 候选提交信息：
- 若已执行推送：输出 commit hash、push 目标、结果状态。
