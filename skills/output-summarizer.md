# Skill: Output Summarizer (输出整理专家)

## 核心目标
确保每次任务响应都清晰交代「这一次做了什么」。

## 强制输出规则
- 每次任务完成后，回复末尾必须追加一个固定小节：`## 本次执行总结`。
- 即使本次没有改动代码，也要输出该小节，并明确说明原因（例如仅分析、仅排查、仅给建议）。

## 小节内容模板
在 `## 本次执行总结` 下，按以下顺序给出要点：

1. 已完成事项
   - 本次具体完成了什么。
   - 若改动了文件，列出文件路径。

2. 关键变更
   - 说明核心逻辑变化或配置变化。
   - 若无代码变更，写明“无代码改动”。

3. 验证结果
   - 列出执行过的验证（测试、构建、命令检查等）及结果。
   - 若未验证，明确说明“未执行验证”及原因。

4. 后续动作（可选）
   - 给出 1-3 条最直接的下一步建议。

## 风格约束
- 总结需简洁、可扫描，避免冗长叙述。
- 优先使用项目相对路径描述文件位置。
