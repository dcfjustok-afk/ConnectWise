# AI 编码使用策略

## 目标

- 让每一步都有明确的 AI 参与点。
- 让工具选择可复用、可解释、可审计。
- 在保证原子步骤的前提下提高执行速度。

## 场景到工具到产物

| 场景 | 推荐工具 | 预期产物 |
|---|---|---|
| 代码库全局摸底 | Copilot Agent Explore | 盘点文档、影响面清单、未知项列表 |
| 单步执行与推理 | Copilot Chat | 当前步骤实现与验收说明 |
| 多文件批量修改 | Copilot Edit | 可审计的最小差异补丁 |
| 规格先行定义 | SDD/OpenSpec | 契约文档、验收标准、边界条件 |
| 专项最佳实践落地 | Skills | 领域级实现建议与检查项 |
| 步骤编排与门禁 | GSD | 原子任务图、Gate 判定与回滚点 |

## 推荐执行流

1. Explore：先收集证据，不先写结论。
2. Spec：把证据转成可执行规格。
3. Code：只改当前步骤，保持最小差异。
4. Test：运行与当前步骤最相关的检查。
5. Review：输出风险和回滚建议。
6. Log：把证据写入会话日志。

## 工具选择决策树

1. 如果任务是调研：优先用 Explore。
2. 如果任务是文档规格：优先用 OpenSpec。
3. 如果任务是实现改动：先 Chat 设计，再 Edit 落盘。
4. 如果任务是专项问题：调用对应 Skill。
5. 如果任务是流程拆解：使用 GSD 输出步骤与门禁。

## 必做规则

- 一次只做一个步骤。
- 每步必须有 AI Coding 参与点。
- 每步必须有验收证据。
- 每步必须可回滚。

## 禁止规则

- 禁止跨步实现。
- 禁止把重构和新功能混在同一步。
- 禁止无证据判定通过。
- 禁止留下未记录的关键假设。
