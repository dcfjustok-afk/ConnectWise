# Step 036 技术栈问答 Q7（测试策略）

问题：迁移测试策略如何设计，才能兜住兼容红线？

## 一、候选方案

- A：单元 + e2e + 协议契约（WS/SSE）分层策略
- B：仅 e2e 回归
- C：仅单元测试

## 二、决策

选择 A：分层测试策略。

## 三、理由

1. 兼容迁移同时涉及 API、会话、实时协议，单一测试层不够。
2. 分层可缩短定位时间：单元查逻辑，e2e查流程，契约查协议。
3. 与 Step 063/064/089/099/124/133/166 路线完全一致。

## 四、风险与缓解

- 风险：初期测试建设成本较高。
- 缓解：按红线优先级增量建设，先覆盖登录、画布、WS/SSE 主链路。

## 五、失败最小修复

- 若短期资源不足，先保证关键 e2e + 协议契约，不允许仅保留冒烟测试。
