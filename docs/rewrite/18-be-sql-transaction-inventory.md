# Step 018 后端 SQL/事务模式盘点

调研目标：梳理 SQL 组织方式、复杂查询与事务边界。

## 一、SQL 组织

- MyBatis XML：`src/main/resources/mapper/*.xml`
- Java 接口：`src/main/java/com/conwise/mapper/*.java`

## 二、查询与更新模式

- 业务以 Mapper + XML 驱动。
- 画布节点与边通常以 JSONB 全量/局部方式更新。
- 分享列表等场景涉及多表查询。

## 三、事务模式

- 当前扫描未发现大量统一事务注解模式。
- 事务边界可能散落在 Service 层实现中。

## 四、风险与迁移提示

- JSONB 高并发更新存在竞态风险，迁移时建议明确乐观锁或版本控制策略。
- 事务边界不清晰会导致迁移后行为漂移，需在实现前补事务清单。

## 五、已确认/待确认

已确认：
- SQL 以 MyBatis XML 为核心组织形式。

待确认：
- 关键写操作的事务边界与回滚策略需在代码重建前细化。
