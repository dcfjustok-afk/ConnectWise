# API 兼容红线规则

## 响应 Envelope

- **必须**：成功响应统一包含 `ok`, `code`, `msg`, `data` 四个字段。
- **必须**：成功响应 `ok=true`，默认 `code=0`，`msg` 为可读文本。
- **必须**：失败响应 `ok=false`，`data=null` 或缺省。
- **禁止**：返回裸数组、裸对象或字段名不一致（例如 `success` 代替 `ok`）。
- **可选**：在不破坏现有字段的前提下追加调试字段（如 `traceId`）。

## 401 行为

- **必须**：未认证请求返回 HTTP 401。
- **必须**：401 响应保持 envelope 结构，便于前端统一拦截。
- **必须**：401 不返回 HTML 页面。
- **禁止**：将未认证场景错误地返回 200 或 500。
- **可选**：在 `msg` 中提供简短可读提示。

## Cookie / Session 行为

- **必须**：后端支持基于 Cookie 的 Session 鉴权。
- **必须**：支持前端 `withCredentials=true` 场景。
- **必须**：登录成功后可在后续请求读取同一会话。
- **禁止**：仅支持 JWT 且不提供 session 兼容层。
- **可选**：增强 cookie 安全属性（如 `HttpOnly`, `SameSite`, `Secure`）。

## 错误码语义

- **必须**：业务错误使用稳定 `code`，同一错误语义不随版本漂移。
- **必须**：关键错误码与旧前端判断逻辑保持一致。
- **必须**：错误码与 HTTP 状态协同，不互相冲突。
- **禁止**：多个不同错误复用同一业务码导致前端误判。
- **可选**：新增错误码时同步补充映射文档。

## 路由前缀与路径

- **必须**：保留 `API` 基础前缀（例如 `/api`）与历史主路径分组（`/user`, `/canvas`, `/share`, `/ai`）。
- **必须**：关键接口路径与方法保持兼容（如 `POST /user/login`, `GET /canvas/:id`）。
- **必须**：WebSocket 与 SSE 路由前缀保持前端可直连。
- **禁止**：无迁移层直接改名路径或改动 HTTP 方法。
- **可选**：新增版本化路径时保留兼容别名。
