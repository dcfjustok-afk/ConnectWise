# Step 030 技术栈问答 Q1（框架）

问题：新后端框架选型应如何确定？

## 一、候选方案

- A：NestJS（Node.js + TypeScript）
- B：Express + 手工分层
- C：继续 Spring Boot

## 二、决策

选择 A：NestJS。

## 三、理由

1. 与目标蓝图一致（JS 全栈，降低前后端上下文切换成本）。
2. 模块化、依赖注入、守卫/拦截器机制天然贴合现有鉴权与统一响应改造需求。
3. 与 Prisma、测试体系（Jest/e2e）生态协同较好，适配 Step 040+ 路线。

## 四、风险与缓解

- 风险：团队对 NestJS 熟悉度不均。
- 缓解：先落地脚手架规范与模板（Phase B 前 10 步内完成）。

## 五、失败最小修复

- 若框架推进受阻，允许短期在 Nest 中采用轻量模块模板，不回退到 Express 或 Spring 双轨并行。
