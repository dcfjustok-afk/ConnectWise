## 七、Phase D/E 追加模板

> 来源：`142-prompt-optimization-3.md` + Retro-4

### P1 兼容审查模板
- **输入**：请审查本阶段对 RL 红线的达成情况，输出 PASS/BLOCK 与证据表
- **输出**：红线对照表 + 阻断项 + 最小修复建议

### P2 稳定性审查模板
- **输入**：请审查重试/熔断/配额/超时行为，给出风险等级
- **输出**：能力核查表 + 命令证据 + 风险处置

### P3 风险收敛模板
- **输入**：请将风险按 CLOSED/DEFERRED/ACCEPTED 分类，并执行最小补丁收敛
- **输出**：风险清单 + 已执行补丁 + 回归结果

### P4 契约测试模板
- **输入**：为接口生成契约测试，覆盖 happy path + error path + edge case
- **输出**：测试文件 + 关键断言 + 执行结果

### P5 Gate 模板
- **输入**：按 Gate-1..Gate-5 给出是否可进入下一阶段
- **输出**：Gate 明细 + 总判定 + 前置条件

### P6 会话日志模板
- **输入**：按 8 字段写入当前 Step 记录，附命令证据
- **输出**：可审计日志条目

### P7 限流测试绕过模板（Retro-4 新增）
- **输入**：E2E 中需要绕过 ThrottlerModule 限流
- **做法**：`overrideProvider(ThrottlerStorage).useValue({ increment: async () => ({ totalHits: 0, timeToExpire: 0 }) })`
- **禁区**：不得 `overrideGuard(ThrottlerGuard)` — 对 APP_GUARD 多提供者绑定无效
- **验收**：所有 E2E 路由不再返回 429

### P8 构造函数变更联动模板（Retro-4 新增）
- **输入**：修改 service/controller 的 constructor 注入列表
- **做法**：修改 constructor → 立即运行关联 spec → 补 mock
- **禁区**：不得跳过 spec 验证
- **验收**：`npm test` 通过
