# 证据留存规范

目标：确保每一步都可审计、可复现、可回滚。

## 每步必备证据

- AI 输出摘要
- 改动文件清单
- 执行命令清单
- 关键命令输出
- 验收检查结果
- 风险与回滚说明

## 证据包结构

每个步骤证据包必须包含：

- 变更了什么：文件与意图
- 如何验证：命令与关键输出
- 为什么通过：对应验收项的证据映射
- 后续风险：明确风险与缓解措施

## 文件路径规则

- 使用工作区相对路径。
- 每个文件仅保留一种规范路径写法。
- 同一步记录中不要混用绝对与相对路径。

## 留存规则

- 所有步骤必须保留命令证据。
- 所有改动文件必须保留 diff 证据。
- 失败或部分通过步骤必须保留失败证据。
- 外部产物未入库时，必须保留截图/日志占位说明。

- 下一步动作：

## 验收映射格式

- 验收项：
- 证据指针：
- 结论：Pass | Block

## 外部产物占位

- 产物类型：screenshot | terminal log | http capture | db query result
- 位置：未提交 / 已提交路径
- 摘要：

## 标记通过前质量检查

- 证据字段无缺失。
- 验收项全部完成证据映射。
- 风险至少包含 1 条缓解动作。
- 回滚说明可直接执行。
