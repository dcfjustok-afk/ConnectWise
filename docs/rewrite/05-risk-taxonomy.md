# 风险分类标准

## 兼容性风险

- API 响应结构漂移
- 错误码语义漂移
- WS/SSE 契约漂移

触发条件：
- 关键接口字段或状态码与前端预期不一致。
- 旧错误码语义被复用或替换。
- 前端事件解析逻辑与新协议不匹配。

止损动作：
- 立即启用兼容层或回退到兼容响应结构。
- 冻结相关接口发布并执行抓包对照。
- 优先修复契约文档与实现差异后再放行。

## 安全风险

- 鉴权绕过
- 输入校验缺失
- 日志敏感信息泄露

触发条件：
- 未登录用户可访问受保护资源。
- 非 owner/edit 角色可执行写操作。
- 日志出现 password/token/cookie 明文。

止损动作：
- 立即阻断发布并回滚当前步骤。
- 补充 Guard/Policy 与 DTO 校验并复测。
- 启用日志脱敏并重跑安全检查。

## 性能风险

- 数据库热点查询退化
- WS 广播放大
- SSE 流泄漏

触发条件：
- 关键接口延迟显著上升或超时。
- 同房间广播导致 CPU/内存异常抖动。
- 长连接数量持续增长且无回收。

止损动作：
- 降级高风险功能并限制并发阈值。
- 增加索引/缓存并优化热点查询。
- 对 WS/SSE 加入连接上限与超时清理。

## 交付风险

- 缺少回滚方案
- 测试证据不完整
- 隐式依赖未追踪

触发条件：
- 发生故障但无法执行回滚步骤。
- Step 记录缺失关键证据字段。
- 环境变量或外部依赖在联调时才暴露。

止损动作：
- 回到当前步骤补全回滚模板与验证清单。
- 阻断 Gate 放行，补齐证据后再评估。
- 增加依赖盘点与启动检查项。

## 严重级别

- P0：阻断上线，涉及数据或安全关键风险
- P1：重大行为风险，合并前必须修复
- P2：中等风险，排期修复
- P3：轻微风险，可进入待办
