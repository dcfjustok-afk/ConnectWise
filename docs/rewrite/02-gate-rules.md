# 步骤门禁规则

目标：为每个原子步骤给出通过、阻断、回滚判定。

## Gate-1 范围锁定

- 仅允许当前步骤范围内变更。
- 禁止无关文件改动。

## Gate-2 证据完整

- 已列出改动文件。
- 已记录命令证据。
- 已给出验收输出。

## Gate-3 质量检查

- 与当前步骤相关的 build/test 有执行结果。
- 若不可执行，必须明确说明原因。

## Gate-4 风险与回滚

- 风险已列出影响与缓解动作。
- 已提供可执行回滚说明。

## Gate-5 日志落盘

- 当前步骤记录已写入会话日志。
- 结果已标记 Passed 或 Blocked。

## 判定矩阵

### 通过（Pass）

- Gate-1 到 Gate-5 全部满足。
- 无未处置 P0/P1 风险。
- 验收证据可复现。

### 阻断（Block）

- 任一门禁缺失关键证据。
- 验收项无法验证。
- 存在未缓解的 P0/P1 风险。

### 需要回滚（Rollback Required）

- 当前步骤触发兼容红线破坏。
- 变更后出现数据一致性风险。
- 当前步骤确认出现安全回归。

## 回滚触发检查清单

- 兼容保护区内 API envelope 不匹配。
- 出现鉴权绕过或越权。
- 迁移造成数据丢失或关系断裂。
- WS/SSE 契约破坏导致客户端不可用。

## 最小修复规则

- 仅修当前步骤范围。
- 禁止顺带新增后续功能。
- 修复后重新执行相关门禁检查。

## 阻断条件

- 出现跨步改动。
- 缺失验收证据。
- 高风险步骤缺失回滚说明。
