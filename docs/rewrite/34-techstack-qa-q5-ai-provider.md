# Step 034 技术栈问答 Q5（AI Provider）

问题：AI Provider 采用什么策略最稳妥？

## 一、候选方案

- A：单 Provider 直连
- B：Provider 抽象层 + 工厂切换（主备）
- C：完全自建模型服务

## 二、决策

选择 B：Provider 抽象层 + 工厂切换。

## 三、理由

1. 迁移期风险不可控，需保留主备切换与熔断降级能力。
2. 与计划中的 Step 126/127/135/136 高度一致，可执行性强。
3. 有利于隔离厂商 SDK 差异，减少业务代码污染。

## 四、风险与缓解

- 风险：抽象层设计过度导致初期交付变慢。
- 缓解：先定义最小接口（generate/associate/generateGraph），后续再扩展。

## 五、失败最小修复

- 若抽象层首版复杂度超预期，先保留单实现 + 统一接口，不改变调用侧契约。
