## 分层回滚动作

### Layer A: API 回滚

1. 回退当前步骤 API 代码补丁。
2. 恢复兼容响应结构与状态码语义。
3. 验证关键接口最小可用集合。

检查项：
- 接口路径与 method 无漂移。
- envelope 与错误码保持兼容。

### Layer B: DB 回滚

1. 停止写入路径，防止继续放大影响。
2. 回退本步 migration 或执行反向修复脚本。
3. 执行数据一致性核对并记录结果。

检查项：
- 主键、外键、索引状态正常。
- 核心数据行数与约束满足预期。

### Layer C: Config 回滚

1. 回退本步配置改动（env、默认值、开关）。
2. 恢复上一个稳定配置快照。
3. 重启服务并执行启动健康检查。

检查项：
- 服务可启动且依赖连接正常。
- 关键配置项值与基线一致。
